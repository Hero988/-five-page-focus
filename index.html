A disciplined first pass

From mixed evidence to fixable tasks

The workflow keeps machine output in its proper place: useful evidence that still needs human judgement.

  1. Choose the sample.Select one to five public pages that represent an important journey or shared templates.
  2. Collect evidence.Run automated checks and capture desktop and narrow-screen states without treating tool output as a verdict.
  3. Review manually.Check keyboard access, focus, structure, forms, images, links, contrast, language, and reflow.
  4. Consolidate.Remove false positives and group repeated component defects by root cause.
  5. Prioritise.Order findings by user impact, reach, and dependencies—not by a made-up score.
  6. Define the re-test.Give each recommendation a specific condition that can be checked after remediation.

Clear boundaries

Useful triage without an inflated promise

A five-page sample can reveal high-confidence patterns and create a practical starting backlog. It cannot establish whole-site conformance or replace evaluation by disabled people and assistive-technology users.

This demonstration is not certification, legal advice, disabled-user research, a complete WCAG conformance evaluation, or a guarantee that every barrier was found.
